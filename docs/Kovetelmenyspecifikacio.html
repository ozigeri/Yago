Bevezetés

A Yago Environment Builder egy Windows Forms alapú asztali alkalmazás, amely a fejlesztők munkáját segíti azáltal, hogy automatizálja és leegyszerűsíti a különböző fejlesztői környezetek létrehozását.

A rendszer célja, hogy a felhasználók könnyedén összeállíthassák a kívánt környezetet különböző verziójú komponensekből (például PHP, Node.js, Composer), sablonokat készíthessenek, illetve új repository-kat generálhassanak a megadott beállítások alapján.

Az alkalmazás célcsoportja elsősorban fejlesztők, akik gyakran dolgoznak több projekt párhuzamos kezelésével, és igénylik a gyors, automatizált környezetbeállítást.

A rendszer grafikus felülete átlátható, három fő funkcionális egységre tagolódik: verziókezelés, repository létrehozás és sablonkezelés.

1. Célok és funkciók

EntryPoint (Kezdőképernyő)

  • A program indításakor megjelenő kezdőfelület.
  • Lehetővé teszi a fő modulok közötti navigációt: Versions, Repository Creator, Templates.
  • A logó megjelenítése.

Versions modul

  • A felhasználó kiválaszthatja a fejlesztői környezet komponenseinek verzióit (PHP, Node.js, Composer).
  • Lehetőség új konfiguráció mentésére sablonként.
  • Korábban mentett sablonok betöltése.
  • Környezet indítása a kiválasztott beállításokkal.

Repository Creator modul

  • Új projekt repository létrehozása megadott környezetbeállítások alapján.
  • A felhasználó kiválaszthatja:
    • Az alkalmazás típusát (pl. React, Laravel stb.)
    • A repository nevét
    • A mentés helyét (helyi mappa)
    • A verziókat (Versions modulból)
    • Lehetőség azonnali Git repository létrehozására
    • Lehetőség IDE megnyitására a repositoryn
  • Repository automatikus generálása a megadott paraméterek alapján.

Templates modul

  • Sablonok listázása (név, verziók).
  • Sablon szerkesztése (PHP, Composer, Node verziók módosítása).
  • Új sablon mentése.
  • Sablon törlése a listából.

2. Érintettek és szerepkörök

Alap felhasználó - Fejlesztő

A fő felhasználói csoport, akik a Yago alkalmazást használják fejlesztői környezetek létrehozására. Feladataik:

  • Környezetverziók kiválasztása és indítása
  • Sablonok létrehozása, szerkesztése és törlése
  • Új repository generálása

3. Nem funkcionális követelmények

  • Gyors reakció a felhasználói műveletekre.
  • Hibakezelés (pl. hibás elérési út, hiányzó verzió).
  • Windows Forms alapú, intuitív grafikus kezelőfelület.
  • Jól elkülöníthető fülek a funkciókhoz.
  • Helyi adattárolás (nincs külső szerver).
  • Egységes vizuális megjelenés és egyszerű navigáció.
  • Visszajelzés minden fő művelet után.

4. Rendszerkörnyezet

  • Nyelv: C#
  • IDE: Visual Studio 2022
  • Keretrendszer: .NET 4.7.2
  • Típus: Windows Forms Application
  • Verziókezelés: Git

Az alkalmazás a sablonokat és beállításokat a Templates.txt fájlban tárolja a gyökérkönyvtárban.

Adatstruktúra: név;php;composer;node

  • Név: Sablon neve (pl. Moodle, Laravel)
  • php: PHP verzió
  • composer: Composer verzió (alapértelmezett: "default")
  • node: Node.js verzió

5. Korlátozások

  • Kizárólag Windows operációs rendszeren futtatható.
  • Szükséges a .NET Runtime 8.0 vagy újabb.
  • Nincs automatikus frissítés.
  • Nem többfelhasználós (lokális használat).

6. Példa felhasználói történetek

  • Mint fejlesztő, szeretném kiválasztani a verziókat a pontos környezet beállításához.
  • Mint fejlesztő, szeretném elmenteni a beállításokat sablonként.
  • Mint fejlesztő, szeretném elindítani a környezetet, hogy azonnal dolgozhassak.
  • Mint fejlesztő, szeretném megadni az adatokat új repository generálásához.
  • Mint fejlesztő, szeretném szerkeszteni a sablonokat a naprakészség érdekében.
  • Mint felhasználó, szeretném törölni a felesleges sablonokat.
  • Mint felhasználó, visszajelzést várok a műveletek sikerességéről.

7. Elfogadási kritériumok

  • Navigáció működik a modulok között.
  • Verziók kiválaszthatók (PHP, Composer, Node.js).
  • Sablonok menthetők és betölthetők.
  • Fejlesztői környezet elindítható.
  • Új repository létrehozható, érvényesség-ellenőrzéssel.
  • Sablonok szerkeszthetők és törölhetők.
  • Visszajelzés megjelenik a műveletek után.

8. Jövőbeli bővítési lehetőségek

  • macOS és Linux támogatás.
  • Komponensek automatikus frissítése.
  • Többfelhasználós mód külön bejelentkezéssel.
  • Felhő alapú szinkronizáció.
  • Előre beállított build- és teszt-scriptek futtatása.

9. Részletes UI/UX követelmények

  • A felület egyszerű, áttekinthető elrendezést használ három fő modullal (Versions, Repository Creator, Templates).
  • Minden modul külön fülön vagy panelen jelenik meg egyértelmű címkékkel.
  • Minden fontos művelethez (mentés, törlés, generálás) külön gomb tartozik.
  • Hibás vagy hiányzó mezők esetén a rendszer vizuális jelzést ad (hibaüzenet).
  • Az alkalmazás modális ablakokat használ a kritikus visszajelzésekhez (pl. sikeres mentés, hibás útvonal).
  • A felhasználói folyamat logikus sorrendben történik (verziók kiválasztása → repository létrehozás → opcionális IDE megnyitás).
  • A sablonok listája rendezhető név szerint.
  • A felület konzisztens ikonokat és egységes színvilágot alkalmaz.

10. Tesztelési követelmények

Unit tesztek

  • A sablonfájl (Templates.txt) olvasásának és írásának tesztelése.
  • A verzióválasztás validációjának tesztelése.

Funkcionális / manuális tesztek

  • Verziók kiválasztása és elindítása.
  • Sablon létrehozása, szerkesztése, törlése.
  • Repository generálása különböző beállításokkal.
  • Repository feltöltése GitHubra.
  • IDE megnyitás funkció tesztelése (ha be van állítva).

Integrációs tesztek

  • A sablonok és a Versions modul közötti kommunikáció ellenőrzése.
  • Repository Creator és Git inicializáló funkciók együttműködése.

Hibakezelés tesztelése

  • Érvénytelen vagy nem létező mappa kiválasztása.
  • Hiányzó verzió esetén megjelenő hibaüzenet.
  • Üres repo név esetén validáció és blokkolás.
